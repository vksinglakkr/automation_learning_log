Screen Par Kya Dikhta HaiWahi 7 Branch — ab JSON nahi, seedha n8n editor ki nazar se

Pichli class me humne shabd seekhe the — Authorization, Bearer, Query Param. Ab dekhte hain ye shabd n8n ke screen par asal me kahan aur kaise dikhte hain.

Jab tum apna workflow khologe, kisi bhi HTTP Request node par click karoge, to right side me ek panel khulta hai. Us panel me sabse upar Method aur URL hote hain, phir Authentication naam ka ek dropdown hota hai. Yehi dropdown decide karta hai ki key kaise bhejna hai. Har branch me is dropdown ki value alag hai — bas yehi asli farak hai.

Yahan Authentication wapas "None" hai — Step 1 jaisa hi, koi credential system use nahi ho raha. Lekin URL me ab peela-hardcoded text nahi hai, balki ek expression {{ $json.apiKey }} hai jo tumhare HTML form se aayi key ko utha leta hai. Matlab: key har baar type nahi karni padti workflow ke andar, par phir bhi wo kahin "save" nahi hoti — sirf us ek request ke liye aati hai aur chali jaati hai.

Yaad rakhne wali baat

Chahe kuch bhi naam ho — Query Auth, Header Auth, Bearer Auth — screen par har baar bas ek dropdown aur ek credential modal hi dikhta hai. Farak sirf itna hai ki modal ke andar konsa khaana bhara hai (Name+Value, ya sirf Token), aur wo jaakar request ke kis hisse (URL / header) me chipakta hai. Ye dono cheez decide karti hai ki server request accept karega ya nahi.
