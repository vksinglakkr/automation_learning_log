## 📖 n8n में Webhook डेटा का "झमेला": पूरी कहानी

मान लीजिए आपके सिस्टम में बाहर से कोई डेटा रूपी **कूरियर (Webhook)** आता है।

कभी-कभार Webhook उस डेटा को एक अंदरूनी डिब्बे में पैक करके भेजता है (जिसे हम `$json.body.method` कहते हैं)। और कभी-कभार सेटिंग या टेस्ट डेटा की वजह से वही डेटा बिना डिब्बे के सीधे बाहर मिल जाता है (जिसे हम `$json.method` कहते हैं)।

यह आपके workflow का कोई बग नहीं है, बल्कि यह Webhook node का एक बदलने योग्य (configurable) स्वभाव है।

---

## 🔍 कहानी में ट्विस्ट: यह कन्फ्यूजन होता क्यों है?

इस कहानी में दो मुख्य पात्र (Settings) हैं जो यह तय करते हैं कि डेटा डिब्बे के अंदर आएगा या बाहर:

1. **डिफ़ॉल्ट स्वभाव (Default Behavior):** Webhook node का नियम है कि वह POST request की बॉडी को `$json.body.*` के अंदर नेस्ट (छिपा) कर रखता है।


2. **"Raw Body" वाला स्विच (Flat Behavior):** अगर आप Webhook node की **Options** सेटिंग में जाकर **"Raw Body"** या "Simplify Response" वाला टॉगल स्विच ऑन/ऑफ कर देते हैं, या टेस्ट के समय डेटा को सीधे टाइप कर देते हैं, तो डेटा बिना डिब्बे के सीधे `$json.method` के रूप में मिलने लगता है।

## 🛠️ समाधान के दो रास्ते: कौन सा चुनें?

इस झमेले से निपटने के लिए हमारे पास **दो तरीके** हैं:

### रास्ता 1: सेटिंग बदलकर रास्ता साफ करना (Webhook Setting Change)

* **कैसे करें:** Webhook node खोलें, नीचे **Options** में जाएं और **"Raw Body"** या बॉडी-पार्सिंग की सेटिंग को अपनी जरूरत के हिसाब से सेट कर दें।


* **फायदा:** workflow में कोई नया node जोड़ने की जरूरत नहीं पड़ती।


* **सावधानी:** अगर भविष्य में किसी बाहरी सर्विस ने डेटा भेजने का तरीका थोड़ा सा भी बदल दिया या आपने टेस्ट डेटा को अलग तरीके से पिन कर दिया, तो `$json.body.*` वाले एक्सप्रेशन फिर से टूट सकते हैं।



---

### रास्ता 2: "सुरक्षा कवच" यानी 'Normalize Input' नोड (सबसे पक्का तरीका)

अगर आप चाहते हैं कि सेटिंग चाहे जो भी हो, आपका workflow कभी न रुके, तो Webhook के तुरंत बाद एक छोटा सा **Edit Fields Node** (पुराना नाम: Set Node) लगा दें।

इस नोड का नाम **"Normalize Input"** रखें और उसमें यह जादुई लाइन लिख दें:

```javascript
method   = {{ $json.body?.method ?? $json.method }}
question = {{ $json.body?.question ?? $json.question }}

```

#### यह सुरक्षा कवच क्यों काम करता है?

* यह एक समझदार गेटकीपर (चौकीदार) की तरह काम करता है।


* यह चेक करता है कि अगर डेटा डिब्बे के अंदर है (`$json.body?.method`), तो वहां से उठा ले; और अगर बाहर खुला पड़ा है (`$json.method`), तो वहां से उठा ले (`??` ऑपरेटर यही काम करता है)।


* इसके बाद, पूरे workflow में आपको कभी सोचना ही नहीं पड़ेगा—आप आंख बंद करके हर जगह सिर्फ `$json.method` और `$json.question` इस्तेमाल कर सकते हैं!
